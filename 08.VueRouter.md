### 路由

+ 基础示例

  ```html
  <div id="app">
      <!--
          路由跳转:
  			to   指定跳转的地址
  				path  指定跳转的路由,和name二选一即可
  				name  指定跳转的路由(路由别名),和path二选一即可
                   query  在路由中添加查询字符串
                   params   路由参数,如果使用路由参数,必须使用name
  
  			tag   改变元素种类
      -->
      <router-link :to="{ name:'home' }" tag="button">首页</router-link>
  
      <!-- 渲染路由下的组件 -->
      <router-view></router-view>
  </div>
  
  
  <script>
      let son1 = {
          template: '<div>首页</div>'
      };
  
      //实例化路由对象
      let router = new VueRouter({
          routes:[
              {//当页面的路由是 home 时,显示 son1组件
                  path:"/home",//路由路径
                  name: "home",//路由的别名
                  component:son1,//需要显示组件
              },
          ]
      });
  
      let vm = new Vue({
          el: "#app",
          router,//注册路由
      });
  </script>
  ```



### 动态路由匹配

```js
我们经常需要把某种模式匹配到的所有路由，全都映射到同个组件。例如，我们有一个 User 组件，对于所有 ID 各不相同的用户，都要使用这个组件来渲染。那么，我们可以在 vue-router 的路由路径中使用“动态路径参数”(dynamic segment) 来达到这个效果：

let User = {
  template: '<div>User</div>'
}

let router = new VueRouter({
  routes: [
    // 动态路径参数 以冒号开头
    { path: '/user/:id', component: User }
  ]
})

现在呢，像 /user/foo 和 /user/bar 等，都将映射到相同的路由。
匹配到一个路由时，参数值会被设置到 this.$route.params，可以在每个组件内使用
```



### 重定向

```js
//实例化路由对象
let router = new VueRouter({
    routes:[
        {path:"/home", component:son1},
        {path:"/stu", component:stu},
        {//当路由不是 home 也不是 stu 时，重定向到not页面
            path:"*",//通配符路由
            redirect:not,//重定向
        }
    ]
});
//当使用通配符路由时，请确保路由的顺序是正确的，也就是说含有通配符的路由应该放在最后
```

### 编程式导航

```js
let vm = new Vue({
    el: "#app",
    router,//注册路由
    methods:{
        goHome(){
            //跳转到 /home?name=xiaoming&age=18
            this.$router.push({
                path:"/home",
                query:{
                    name:"xiaoming",
                    age:"18"
                }
            })
        },
        goStu(){
            //跳转到 /stu/xiaoming
            this.$router.push({
                name:"stu",
                params:{
                    id:"xiaoming"
                }
            })
        },
        goBack(){
            //返回上一页
            this.$router.go(-1)
        },
    }
});
```



### 响应路由参数的变化

```HTML
当使用路由参数时，例如从 /user/foo 导航到 /user/bar，原来的组件实例会被复用。因为两个路由都渲染同个组件，比起销毁再创建，复用则显得更加高效。不过，这也意味着组件的生命周期钩子不会再被调用。

//解决方法1，监听路由
let User = {
  template: '...',
  watch: {
    '$route' (to, from) {
      // 对路由变化作出响应...
    }
  }
}


//解决方法2，使用 2.2 中引入的 beforeRouteUpdate 导航守卫：
let User = {
  template: '...',
  beforeRouteUpdate (to, from, next) {
    // react to route changes...
    // don't forget to call next()
  }
}


//解决方案3，直接在路由视图标签上打上标记
<router-view :key='Math.random()'></router-view>
```
